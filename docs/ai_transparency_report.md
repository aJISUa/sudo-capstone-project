## 1. 어떤 AI를, 어떤 작업에 사용했는가

### ChatGPT

* 프로젝트 아이디어 정리
* 발표 흐름 및 스토리 구성
* 사용자 인터뷰 질문 초안 작성

### Gemini

* 사용자 시나리오 초안 작성
* UX 흐름 및 기능 방향 탐색
* 식단 분석·개인화 피드백 기능 구현 가능성 조사

### Claude

* 시스템 아키텍처 아이디어 탐색
* 기능 구성 및 화면 흐름 구조화

### Figma Make

* UI/UX 와이어프레임 생성
* 레이아웃 초안(Draft) 제작

### Claude Code

* Flutter UI 구현 참고
* 반복 UI 코드 및 레이아웃 예시 활용


## 2. AI 제안을 수정하거나 기각한 사례

### 사례 1 — 서비스 방향 수정

#### AI 제안

* 운동 기록·칼로리 계산 중심의 일반 헬스케어 앱 구조 제안

#### 팀 판단

* 단순 피트니스 앱보다
  **“불규칙한 생활 속 만성질환 위험 관리”**에 집중하는 방향이 더 중요하다고 판단

#### 최종 결정

* 식단·생활 패턴·건강 요약 중심 UX 흐름으로 화면 구조 수정



### 사례 2 — 기능 범위 조정

#### AI 제안

* 챗봇·자동 추천·분석 기능 등 다양한 AI 기능 동시 구현 제안

#### 팀 판단

* 기능 수보다 핵심 사용자 흐름과 UI 안정성 확보가 우선이라고 판단

#### 최종 결정

* Flutter 기반 UI와 사용자 시나리오 중심 MVP 형태로 우선 개발



## 3. AI가 틀렸거나 못 믿었던 순간

### 사례 1 — 불필요한 기능 과다 제안

#### 문제 상황

* 실제 서비스 흐름과 맞지 않는 탭·화면 구성이 포함되는 경우 발생

#### 팀 대응

* 핵심 사용자 흐름 중심으로 필요한 기능만 남기도록 화면 구조 재구성



### 사례 2 — 미구현 기능 포함 제안

#### 문제 상황

* 아직 구현되지 않은 기능까지 실제 서비스 구조처럼 제안하는 경우 발생

#### 팀 대응

* 구현 완료 기능과 아이디어 단계 기능을 구분하여 README 및 발표 자료 재정리


## 4. AI 없이 우리가 직접 한 것

* “불규칙한 생활을 하는 2030 위험군”이라는 타겟 설정 직접 결정

### 발표 시나리오 및 UX 흐름 설계

* 사용자 행동 흐름 기반 발표 구조 직접 설계

### GitHub 협업 및 프로젝트 관리

* 브랜치 전략·역할 분담·PR 운영 방식 직접 관리
